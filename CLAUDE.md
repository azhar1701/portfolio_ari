# Portfolio Design Guidelines

## Design Context

### Users
- **Technical Auditors**: Recruiters and fellow engineers looking for technical depth, precise GIS data, and irrigation engineering expertise.
- **Stakeholders & Clients**: Potential clients or project managers looking for innovative solutions, professional reliability, and clear project impact.

### Brand Personality
- **Professional**: Clean, authoritative, and trustworthy.
- **Technical**: Data-driven, precise (GIS/Engineering focus), and structurally sound.
- **Innovative**: Utilizing modern tech (Gemini AI, Tailwind v4) to solve traditional engineering communication challenges.

### Aesthetic Direction
- **Visual Tone**: Precision-engineered aesthetics. Think clean lines, subtle technical grids, and high-quality data visualizations.
- **Theme**: Full support for both light and dark modes, defaulting to system preference but ensuring high-contrast readability in both.
- **Flair**: Modern "dynamic flair" using smooth AOS animations, interactive maps (Leaflet), and responsive, fluid layouts.

### Design Principles
1. **Precision First**: Every UI element should feel intentional and precise, reflecting the nature of irrigation and water resources engineering.
2. **Dynamic Professionalism**: Use animations and transitions (dynamic flair) to make the professional content feel alive and modern, not static.
3. **Data-Rich but Accessible**: Present complex technical data (GIS, project stats) in a way that is readable for broad audiences while remaining deeply informative for technical peers.
4. **Adaptive Excellence**: The interface must remain "impeccable" across all device sizes and theme preferences (light/dark).
